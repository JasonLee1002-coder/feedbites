菜單自定義分類

讓店長能自由管理菜單分類，取代現行 8 個硬編碼預設分類。分類扁平單層，各店獨立，上線自動繼承現有分類。

DB Schema

新增 dish_categories table：

dish_categories(
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  store_id   UUID NOT NULL REFERENCES stores(id) ON DELETE CASCADE,
  name       TEXT NOT NULL,
  position   INT NOT NULL DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
  UNIQUE(store_id, name)
)
CREATE INDEX ON dish_categories(store_id, position ASC);

dishes.category 保持 TEXT，存分類名稱（非 id），前端顯示免 join。

RLS：service role 全開（與現有模式一致）。

遷移策略

上線時對每家店自動遷移現有分類：

INSERT INTO dish_categories (store_id, name, position)
SELECT DISTINCT store_id, category,
  ROW_NUMBER() OVER (PARTITION BY store_id ORDER BY category) - 1
FROM dishes
WHERE category IS NOT NULL AND category != ''
ON CONFLICT (store_id, name) DO NOTHING;

API Endpoints

MethodPath說明
GET/api/menu/categories取該店分類清單（position 排序）
POST/api/menu/categories新增分類
PATCH/api/menu/categories/[id]改名 / 改 position
DELETE/api/menu/categories/[id]刪除（dishes 不動，顯示「未分類」）

所有端點：createServerSupabase() 驗 auth → getSelectedStore() 取 store_id → createServiceSupabase() 操作 DB。

前端 — 設定頁

路由：/dashboard/settings/categories（整合到現有設定頁新 tab）

  • 顯示分類清單（position 排序）
  • 拖拉排序（@dnd-kit/sortable，已在 deps 中）
  • 新增分類（input + 送出）
  • 重新命名（inline edit）
  • 刪除（確認 modal：「X 道菜將顯示為『未分類』」）

前端 — 菜色表單

現有新增/編輯菜色的分類欄位改為：

  • <select> 列出所有 dish_categories（position 排序）
  • 最底部固定選項「＋ 新增分類...」
  • 點擊後顯示 inline input，送出後同步 POST /api/menu/categories 並選中新分類

Error Handling

  • 刪除有菜色使用中的分類：後端允許刪除，前端提示「X 道菜將顯示為未分類」
  • 分類名重複：後端 UNIQUE constraint 回 409，前端提示「此分類名稱已存在」
  • 新店無分類：菜色表單允許直接 inline 建立，不阻擋操作

Scope

✅ In Scope

  • dish_categories table + RLS
  • 遷移 SQL（一次性）
  • 4 個 API 端點
  • 設定頁分類管理 UI（含拖拉排序）
  • 菜色表單分類欄位改造

  • 菜單頁依分類 tab 顯示
